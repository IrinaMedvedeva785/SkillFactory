#Сценарий работы с оформлением заказа


#####Это сценарий, который описывает как пользователь будет действовать при подтверждении заказа


~~После нажатия кнопки “Далее” пользователь проверяет время и адрес доставки, указывает способ оплаты и переходит к подтверждению заказа.~~


**Основной сценарий:**

После нажатия кнопки “Далее” система отображает данные: все выбранные позиции с сайта, дата и время доставки, адрес доставки, способ оплаты, кнопку “Оформить заказ”. 
Пользователь проверяет данные.


**Альтернативы:**

~~Пользователь захотел добавить другой адрес доставки, система выводит кнопку добавления нового адреса и после нажатия на нее - поля для заполнения адреса, 
а затем кнопку сохранить~~

1. При желании пользователь может добавить еще один адрес доставки, для этого:

+ Система выводит кнопку добавления нового адреса

+ Пользователь нажимает кнопку

+ Система выводит поля для заполнения адреса

+ Пользователь заполняет данные и нажимает кнопку “Сохранить”

2. Пользователь решил изменить время доставки - система отображает доступные временные интервалы

3. Пользователь хочет выбрать другой способ оплаты, система выводит все доступные способы и пользователь выбирает тот, 
который ему больше всего нравится (система должна запоминать последний выбранный способ оплаты)


**Исключения:**

1. Банк не принимает в данное время безналичную оплату, поэтому пользователю выводится информация о том, что можно оплачивать только наличными. 
Система отображает только один доступный способ оплаты и пояснения по нему

2. Пользователь передумал делать заказ и закрыл страницу
